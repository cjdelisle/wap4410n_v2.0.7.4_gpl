VLAN and QoS


VLAN Enabling VLAN to enable 802.1p and 802.1q protocol. Access Point will bridge wireless and wired traffic with VLAN tagged packets. Each SSID will be placed in its own virtual network (VLAN) and traffic will not be passed from one VLAN to the other.
 
Default VLAN ID Enter the default VLAN ID number (1 - 4094), the default value is 1. The default VLAN number should match with your switch's settings. For example, SRW2024 has Trunk port mode which set default VLAN (PVID) to 1 untagged, while General port mode can set PVID to any VLAN either tagged or untagged.
 
VLAN Tag Set the tagging option for default VLAN ID. This has to match with your switch's settings. The default is untagged.
 
AP management VLAN When the VLAN option is enabled, the value entered (VLAN ID) in this field defines the VLAN that connects to the Access Point. The VLAN should be reachable from the wired side in order to use web-based utility. To access web-based utility from wireless side, the SSID needs to map to the same VLAN. Remember to enable wireless web access on Administration/Management Tab. The default value is 1.
 
VLAN Tag over WDS
  		Set the Enabled option for packets over WDS. The default is Disabled.
 
VLAN ID Select a VLAN ID (1 - 4094) for the SSID where you want to map the traffic to on the wired side. The wireless traffic will not carry VLAN information. Multiple SSIDs can map to the same VLAN on the wired side.

    Note: To use wireless security which requires RADIUS server (802.1X) authentication, the VLAN ID needs to be the same as AP management VLAN ID. We will remove this restriction in future releases.
 
Priority Assign VLAN ID and VLAN Priority to the specific SSID, Access Point will lead Ethernet to receive / transmit VLAN tagged packets from / to another device.
 
WMM WMM (Wi-Fi Multimedia) is a component of the IEEE 802.11e wireless LAN standard for quality of service (QoS). It specifically supports priority tagging and queuing. Click the check box to Enabled WMM support.
