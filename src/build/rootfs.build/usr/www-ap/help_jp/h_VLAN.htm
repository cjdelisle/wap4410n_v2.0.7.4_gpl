VLAN および QoS


[VLAN] VLAN を有効にして、802.1p および 802.1q のプロトコルを有効にします。アクセス ポイントは、VLAN タグ付きのパケットで、ワイヤレスおよび有線のトラフィックをブリッジします。各 SSID がその仮想ネットワーク（VLAN）に配置され、トラフィックは 1 つの VLAN から別の VLAN に渡されません。
[デフォルトVLAN ID] デフォルトの VLAN ID 番号（1 ～ 4094）を入力します。デフォルト値は 1 です。デフォルトの VLAN 番号は、スイッチの設定と一致する必要があります。たとえば、SRW2024 のトランク ポート モードはデフォルトの VLAN（PVID）を 1 のタグなしに設定しますが、一般ポート モードでは PVID をタグ付きまたはタグなしの任意の VLAN に設定できます。
[VLANタグ] デフォルトの VLAN ID のタグ オプションを設定します。この設定は、スイッチの設定と一致する必要があります。デフォルトではタグなしになります。
[AP管理VLAN] VLAN オプションを有効にした場合、このフィールドに入力した値（VLAN ID）によって、アクセス ポイントに接続する VLAN が定義されます。Web ベースのユーティリティを使用するには、有線の側から VLAN に接続できる必要があります。ワイヤレスの側から Web ベースのユーティリティにアクセスするには、SSID が同じ VLAN にマップされている必要があります。[各種管理] > [管理] タブで必ずワイヤレス Web アクセスを有効にしてください。デフォルト値は 1 です。
[WDSによるVLANタグ]
WDS で送受信されるパケット用に有効になるオプションを設定します。デフォルトでは無効になります。
[VLAN ID] 有線側でトラフィックをマップする先の SSID 用に VLAN ID（1 ～ 4094）を選択します。ワイヤレス トラフィックでは VLAN 情報が伝えられません。複数の SSID を有線側の同じ VLAN にマップできます。

注：RADIUS サーバ（802.1X）認証が必要なワイヤレス セキュリティを使用するには、VLAN ID が AP 管理の VLAN ID と同じである必要があります。将来のリリースではこの制約が削除される予定です。
[プライオリティ] VLAN ID および VLAN プライオリティを特定の SSID に割り当てると、アクセス ポイントはイーサネットが他のデバイスとの間で VLAN タグ付けされたパケットをやり取りするように制御します。
[WMM] Wi-Fi Multimedia（WMM）は、Quality of Service（QoS）用の IEEE 802.11e ワイヤレス LAN 規格のコンポーネントです。WMM はプライオリティのタグ付けとキューイングをサポートしています。WMM サポートを有効にするには、チェックボックスをオンにします。
